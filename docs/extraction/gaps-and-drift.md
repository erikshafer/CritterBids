# Gaps and Drift Register

Populated in Phase 4. Strictly factual. Each entry: the claim and its source (cited), the reality and its source (cited), and a classification.

Sections:

1. **Doc-vs-code drift** — an authoritative-looking doc contradicts code or a superseding ADR.
2. **Declared-but-not-built** — capabilities or whole BCs the docs describe that have no code.
3. **Declared-but-not-wired** — events, commands, or integration messages named in `Contracts`, vision doc, or a dossier but with no emitter, no consumer, or no registration.

This file does not evaluate. Drift is an observed fact. Repair, blame, and "what should happen" belong in `lessons.md` (and only there, generalized).
